# Videos

## Video guidelines

Docker 文档中很少使用视频。如果使用，视频应作为书面文本的补充，而不应作为文档的唯一格式。与书面文本甚至截图相比，视频的制作时间更长，维护也更困难，因此在添加视频之前，请考虑以下几点：

- 您能否证明使用视频有明确的客户需求？
- 视频是否提供了新的内容，而不是直接照读或重新利用官方文档？
- 如果视频包含可能经常变化的用户界面，您是否有维护计划以保持视频内容的时效性？
- 视频的[语气和语调](../style/voice-tone.md)是否与文档的其余部分保持一致？
- 您是否考虑过其他选项，例如截图或澄清现有文档？
- 视频的质量是否与 Docker 其他文档的质量相当？
- 视频是否可以链接或嵌入到网站中？

如果满足以上所有标准，您可以在创建视频以添加到 Docker 文档之前，参考以下最佳实践。

### Best practices

- 确定视频的目标受众。视频是为初学者提供的广泛概述，还是为高级用户设计的深入技术流程讲解？
- 视频时长应少于 5 分钟。请考虑视频需要多长时间才能恰当地解释主题，如果视频需要超过 5 分钟，请考虑使用文本、图表或截图代替。这些形式更便于用户扫描以查找相关信息。
- 视频应遵守与文档其余部分相同的可访问性标准。
- 通过编写脚本（如果有旁白）、确保不显示多个浏览器和 URL、模糊或裁剪掉任何敏感信息，以及在不同浏览器或屏幕之间使用平滑过渡，来确保视频质量。

视频不托管在 Docker 文档仓库中。要添加视频，您可以[链接到](./links.md)托管的内容，或使用 [iframe](#iframe) 进行嵌入。


## iframe

要在文档页面上嵌入视频，请使用 `<iframe>` 元素：

```html
<iframe
  class="border-0 w-full aspect-video mb-8"
  allow="fullscreen"
  title=""
  src=""
  ></iframe>
```

## asciinema

`asciinema` 是一个用于录制终端会话的命令行工具。这些录制内容可以嵌入到文档网站上。它们类似于 `console` 代码块，但由于它们是可播放和可拖动的视频，在某些情况下比静态代码块更有用。`asciinema` "视频"中的文本也可以被复制，这使得它们更加实用。

在以下情况下考虑使用 `asciinema` 录制：

- 终端命令的输入/输出对于静态示例来说太长（您也可以考虑截断输出）
- 您要展示的步骤可以通过几个命令轻松演示
- 需要同时查看命令的输入和输出

要创建 `asciinema` 录制并将其添加到文档中：

1. [安装](https://docs.asciinema.org/getting-started/) `asciinema` CLI
2. 运行 `asciinema auth` 以配置您的客户端并创建账户
3. 使用 `asciinema rec` 开始新录制
4. 运行演示的命令，并使用 `<C-d>` 或 `exit` 停止录制
5. 将录制内容上传到 <asciinema.org>
6. 使用 <asciinema.org> 上的 **Share** 按钮，通过 `<script>` 标签嵌入播放器
