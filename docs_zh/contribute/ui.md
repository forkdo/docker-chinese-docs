在编写涉及按钮、字段、菜单、对话框或其他用户界面（UI）元素的文档时，请使用本指南。本指南说明了如何格式化 UI 术语、编写以任务为导向的说明，以及一致且清晰地引用常见的 UI 模式。

## 格式化 UI 元素名称

使用粗体格式表示 UI 元素的可见名称：

- 按钮
- 对话框
- 窗口
- 选项卡
- 菜单项
- 列表项
- 表单标签
- 章节标题

例如：

*选择 **Create**，然后填写 **Name** 字段。*

除非产品名称或功能在 UI 中以完全相同的标签形式出现，否则不要加粗。

### 大小写

- 遵循 UI 中的大小写格式。
- 如果 UI 标签全部大写或大小写不一致，请在文档中使用句子大小写以提高可读性。

## 编写以任务为导向的说明

在可能的情况下，根据用户想要完成的任务来指导他们，而不仅仅是告诉他们应该选择什么。这使得文档更具目标导向性，并能更好地适应 UI 变化。

| 推荐做法                        | 避免的做法                             |
|----------------------------------|-------------------------------------------|
| 展开 **Advanced options** 部分。 | 选择 zippy 以展开 **Advanced options** 部分。 |
| 为容器选择基础镜像。 | 选择下拉菜单并选择某个选项。    |


## 使用正确的 UI 元素介词

根据所引用的 UI 元素类型选择合适的介词。

| 介词 | 用于...                    | 示例 |
|-------------|--------------------------------|---------|
| **in**      | 对话框、字段、列表、菜单、窗格、窗口 | 在 **Name** 字段中输入项目名称。 |
| **on**      | 页面、选项卡、工具栏          | 在 **Settings** 选项卡上，选择 **General**。 |


## 使用一致的 UI 元素术语

在引用 Docker 产品中的元素时，请使用以下标准术语：

| 首选术语      | 引用...                     |
|---------------------|----------------------------------------------|
| **button**          | 可点击的操作元素（例如，**Start**） |
| **field**           | 输入文本或选择值的位置      |
| **menu** / **menu item** | 下拉菜单或导航选项         |
| **drop-down**       | 下拉菜单项                             |
| **context switcher** | 用于切换云模式的特定功能          |
| **tab**             | 窗口或页面内的可选视图    |
| **dialog**          | 用于确认或选项的弹出窗口  |
| **section**         | 页面上的逻辑内容分组      |
| **list** / **list item** | 可滚动的选择条目列表 |
| **toggle**  | 二元控制（开/关）               |
| **checkbox**        | 多选控件                      |
| **tooltip**         | 悬停时显示的文本                  |

最后，与其说“点击控件”，不如说“选择 **Create** 按钮”。
