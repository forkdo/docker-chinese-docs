## 视频指南

Docker 文档中很少使用视频。当使用视频时，视频应作为书面文本的补充，而不是作为文档的唯一格式。视频的制作和维护可能比书面文本甚至截图更耗时，因此在添加视频之前请考虑以下几点：

- 您能否证明使用视频有明确的客户需求？
- 视频是否提供了新内容，而不是直接阅读或重新利用官方文档？
- 如果视频包含可能经常变化的用户界面，您是否有维护计划来保持视频内容的时效性？
- 视频的[语气和语调](../style/voice-tone.md)是否与文档的其余部分保持一致？
- 您是否考虑过其他选择，例如截图或澄清现有文档？
- 视频质量是否与 Docker 文档的其余部分保持一致？
- 视频是否可以从网站上链接或嵌入？

如果满足以上所有标准，您可以在创建要添加到 Docker 文档的视频之前参考以下最佳实践。

### 最佳实践

- 确定视频的受众。视频是面向初学者的广泛概述，还是面向高级用户的深入技术流程演示？
- 视频长度应少于 5 分钟。请考虑清楚解释主题所需的视频长度，如果视频需要超过 5 分钟，请考虑使用截图或澄清现有文档。这些方式更便于用户快速扫描找到相关信息。
- 视频应遵循与文档其余部分相同的无障碍标准。
- 通过编写脚本（如果有旁白）、确保不显示多个浏览器或 URL、模糊或裁剪敏感信息以及在不同浏览器或屏幕之间使用平滑过渡来确保视频质量。

视频不托管在 Docker 文档仓库中。要向文档页面添加视频，您可以[链接](./links.md)到托管内容，或使用[iframe](#iframe)嵌入。

## iframe

要在文档页面嵌入视频，请使用 `<iframe>` 元素：

```html
<iframe
  class="border-0 w-full aspect-video mb-8"
  allow="fullscreen"
  title=""
  src=""
  ></iframe>
```

## asciinema

`asciinema` 是一个用于录制终端会话的命令行工具。录制内容可以嵌入到文档网站中。这些录制内容与 `console` 代码块类似，但由于它们是可播放和可拖拽的视频，在某些情况下比静态代码块更有用。`asciinema` "视频"中的文本也可以复制，这使得它们更加实用。

如果符合以下情况，请考虑使用 `asciinema` 录制：

- 终端命令的输入/输出太长，不适合静态示例（您也可以考虑截断输出）
- 您要展示的步骤可以通过几个命令轻松演示
- 需要同时看到命令的输入和输出

要创建 `asciinema` 录制并添加到文档中：

1. [安装](https://docs.asciinema.org/getting-started/) `asciinema` CLI
2. 运行 `asciinema auth` 配置您的客户端并创建账户
3. 使用 `asciinema rec` 开始新录制
4. 运行演示命令，然后使用 `<C-d>` 或 `exit` 停止录制
5. 将录制上传到 <asciinema.org>
6. 使用 <asciinema.org> 上的**分享**按钮，通过 `<script>` 标签嵌入播放器
