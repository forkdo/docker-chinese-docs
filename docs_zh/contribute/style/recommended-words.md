为了帮助确保文档的一致性，技术写作团队推荐使用以下措辞选择。

#### & (and 符号)

不要在标题、正文、导航、UI 文本或目录中使用 `&` 代替 `and`。

#### above

尽量避免在文档页面中使用 `above` 来指代示例或表格。如果需要，使用 `previous`。

例如：

_在前面的示例中，狗有跳蚤。_

#### account name

不要使用。请改用 `username`。

#### admin

首次使用时请写全 `administrator`。如果它是 UI 标签或其他元素的名称，则使用 `admin`。

#### allows

不要使用。请改用 `lets`。

#### as of this writing

避免使用，因为写作本身已经暗示了这个短语。这个短语也可能过早地透露产品或功能策略，或不恰当地暗示产品或功能可能会改变。

#### below

尽量避免在文档页面中使用 `below` 来指代示例或表格。如果需要，使用 `following`。

例如：

_在下面的示例中，狗有跳蚤。_

#### checkbox

使用一个词 `checkbox`。不要使用 `check box`。

你使用 `select`（而不是 `check` 或 `enable`）和 `clear`（而不是 `deselect` 或 `disable`）复选框。

#### click

不要使用 `click`。请改用 `select` 来表示按钮、链接、菜单项和列表。

`select` 适用于更多设备，而 `click` 更特定于鼠标。

#### currently

在谈论产品或其功能时不要使用 `currently`。文档描述的是产品当前的状态。

#### disable

不要使用 `disable`。暗示残疾是一种不理想或负面的状态。

请改用 `turn off` 或 `toggle off`。

在某些技术功能中，开发团队使用 `disable`，在这种情况下，使用该术语是可以的。

#### earlier

在谈论版本号时使用 `earlier`。

使用：

_In Docker Desktop 4.1 and earlier._

而不是：

_In Docker Desktop 4.1 and lower._

#### easy, easily

对你来说容易的事情对别人来说可能不容易。尝试从句子中删除这个词，因为通常可以在没有它的情况下传达相同的含义。

#### e.g.

不要使用。请改用 `for example` 或 `such as` 等短语。

#### enable

不要使用 `enable`。暗示残疾是一种不理想或负面的状态。

请改用 `turn on` 或 `toggle on`。

在某些技术功能中，开发团队使用 `enable`，在这种情况下，使用该术语是可以的。

#### execute

尽量避免使用。请改用 `run`。

#### later

在谈论版本号时使用 `later`。

使用：

_In Docker Desktop 4.1 and later._

而不是：

_In Docker Desktop 4.1 and higher…_ 或 _In Docker Desktop 4.1 and above…_

#### please

在正常解释产品使用方法时不要使用 `please`，即使你在解释一个困难的任务。也不要使用 `please note` 短语。

#### register

在谈论创建账户时使用 `sign up` 而不是 `register`。

#### repo

不要使用。请改用 `repository`。

#### respectively

避免使用 `respectively`，而是更精确地表达。

#### scroll

避免使用。如果上下文清晰，请使用诸如 _move through_ 或 _navigate to_ 等动词短语。

#### sign in

在谈论登录时使用 `sign in` 而不是 `sign on`、`signon`、`log on`、`logon` 或 `log in`、`login`。如果用户界面使用不同的词，请使用那些词。

使用 `sign in to` 而不是 `sign into`。

#### sign up

在谈论创建账户时使用 `sign up` 或 `create account` 而不是 `register`。

#### tab versus view

在指代 UI 中的主要部分时使用 `view`。在指代 UI 中的子部分时使用 `tab`。

例如，在 Docker Desktop 中，**Images** view 和 **Local** tab。

#### toggle

你打开或关闭一个 toggle。例如：

_Turn on the dark mode toggle._

#### upgrade

在描述更高的订阅层级时使用 `upgrade`。

#### vs

不要使用 `vs` 或 `vs.` 作为 versus 的缩写；请改用未缩写的 `versus`。

#### we

尽量避免使用 `we`，而是专注于用户如何在 Docker 中执行某项操作。

使用：

_Use widgets when you have work you want to organize._

而不是：

_We created a feature for you to add widgets._

#### wish

不要使用。请改用 `want`。
