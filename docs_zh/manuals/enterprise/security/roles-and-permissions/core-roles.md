核心角色是 Docker 内置的角色，具有预定义的权限集。
本文档概述了 Docker 的核心角色以及每个角色的权限。

## 什么是核心角色？

Docker 组织有三种核心角色：

- **成员 (Member)**：非管理员角色，具有基本访问权限。成员可以查看组织中的其他成员，并从有访问权限的仓库中拉取镜像。
- **编辑者 (Editor)**：具有部分管理访问权限。编辑者可以创建、编辑和删除仓库，还可以管理仓库的团队权限。
- **所有者 (Owner)**：具有完全的管理访问权限。所有者可以管理所有组织设置，包括仓库、团队、成员、账单和安全功能。

> [!NOTE]
>
> 公司所有者具有与组织所有者相同的组织管理权限，但某些内容和注册表权限是公司所有者不具备的（例如，仓库的拉取/推送）。更多信息请参阅 [公司概述](/admin/company/)。

### 内容和注册表权限

这些权限适用于整个组织，包括组织命名空间中的所有仓库。

| 权限                                            | 成员 | 编辑者 | 所有者 |
| :---------------------------------------------------- | :----- | :----- | :---- |
| 浏览镜像和扩展                                    | ✅     | ✅     | ✅    |
| 收藏、点赞、投票和评论内容                          | ✅     | ✅     | ✅    |
| 拉取镜像                                          | ✅     | ✅     | ✅    |
| 创建和发布扩展                                    | ✅     | ✅     | ✅    |
| 成为已验证、官方或开源发布者                       | ❌     | ❌     | ✅    |
| 编辑和删除发布者仓库徽标                          | ❌     | ✅     | ✅    |
| 作为发布者查看内容互动                            | ❌     | ❌     | ✅    |
| 创建公开和私有仓库                                | ❌     | ✅     | ✅    |
| 编辑和删除仓库                                    | ❌     | ✅     | ✅    |
| 管理标签                                          | ❌     | ✅     | ✅    |
| 查看仓库活动                                      | ❌     | ❌     | ✅    |
| 设置自动化构建                                    | ❌     | ❌     | ✅    |
| 编辑构建设置                                      | ❌     | ❌     | ✅    |
| 查看团队                                          | ✅     | ✅     | ✅    |
| 为仓库分配团队权限                                | ❌     | ✅     | ✅    |

当您将成员添加到团队时，可以授予超出其组织角色的额外仓库权限：

1. 角色权限：组织范围应用（成员或编辑者）
2. 团队权限：特定仓库的额外权限
