Docker Hardened Images (DHI) 是一组经过精心挑选的安全、可用于生产的容器镜像。本文档说明如何探索可用的 DHI 仓库、查看镜像元数据、检查变体详细信息以及了解提供的安全证明。使用这些信息来评估和选择适合您应用程序的镜像变体。

## 探索 Docker Hardened Images

要探索 Docker Hardened Images (DHI)：

1. 访问 [Docker Hub](https://hub.docker.com) 并登录。
2. 选择 **My Hub**。
3. 在命名空间下拉菜单中，选择您有权访问 DHI 的组织。
4. 选择 **Hardened Images** > **Catalog**。

在 DHI 页面上，您可以浏览镜像、搜索镜像或按类别筛选镜像。

## 查看仓库详细信息

要查看仓库详细信息：

1. 访问 [Docker Hub](https://hub.docker.com) 并登录。
2. 选择 **My Hub**。
3. 在命名空间下拉菜单中，选择您有权访问 DHI 的组织。
4. 选择 **Hardened Images** > **Catalog**。
5. 在 DHI 目录列表中选择一个仓库。

仓库详细信息页面提供以下内容：

 - 概述：镜像的简要说明。
 - 指南：关于如何使用镜像和迁移现有应用程序的多个指南。
 - 标签：选择此项以 [查看镜像变体](#view-image-variants)。
 - 安全摘要：选择标签名称以查看快速安全摘要，包括包数量、已知漏洞总数和 Scout 健康分数。
 - 最近推送的标签：最近更新的镜像变体列表及其最后更新时间。
 - 镜像到仓库：选择此项将镜像镜像到您组织的仓库中以使用它。只有组织所有者才能镜像仓库。
 - 在仓库中查看：仓库镜像后，您可以选择此项查看仓库镜像的位置，或将镜像镜像到另一个仓库。

## 查看镜像变体

标签用于标识镜像变体。镜像变体是同一应用程序或框架的不同构建，针对不同用例进行了定制。

要探索镜像变体：

1. 访问 [Docker Hub](https://hub.docker.com) 并登录。
2. 选择 **My Hub**。
3. 在命名空间下拉菜单中，选择您有权访问 DHI 的组织。
4. 选择 **Hardened Images** > **Catalog**。
5. 在 DHI 目录列表中选择一个仓库。
6. 选择 **Tags**。

**Tags** 页面提供以下信息：

- 标签：所有可用标签的列表，也称为镜像变体。
- 合规性：列出相关的合规标识。例如，`FIPS` 或 `STIG`。
- 发行版：变体基于的发行版。例如，`debian 12` 或 `alpine 3.21`。
- 包管理器：变体中可用的包管理器。例如，`apt`、`apk` 或 `-`（无包管理器）。
- Shell：变体中可用的 shell。例如，`bash`、`busybox` 或 `-`（无 shell）。
- 用户：容器运行的用户。例如，`root`、`nonroot (65532)` 或 `node (1000)`。
- 最后推送：镜像变体最后推送的天数。
- 漏洞：基于严重程度的变体中的漏洞数量。
- 健康：变体的 Scout 健康分数。选择分数图标以获取更多详细信息。

> [!NOTE]
>
> 与 Docker Hub 上的大多数镜像不同，Docker Hardened Images 不使用
> `latest` 标签。每个镜像变体都使用完整的语义版本标签（例如，`3.13`、`3.13-dev`）发布，并保持更新。如果您需要固定到特定的镜像版本以确保可重现性，可以按 [摘要](../core-concepts/digests.md) 引用镜像。

## 查看镜像变体详细信息

要探索镜像变体的详细信息：

1. 访问 [Docker Hub](https://hub.docker.com) 并登录。
2. 选择 **My Hub**。
3. 在命名空间下拉菜单中，选择您有权访问 DHI 的组织。
4. 选择 **Hardened Images** > **Catalog**。
5. 在 DHI 目录列表中选择一个仓库。
6. 选择 **Tags**。
7. 在表格中选择镜像变体的标签。

镜像变体详细信息页面提供以下信息：

- 包：镜像变体中包含的所有包的列表。此部分包括每个包的详细信息，包括其名称、版本、发行版和许可信息。
- 规范：镜像变体的规范包括以下关键详细信息：
   - 源和构建信息：镜像基于此处找到的 Dockerfile 和 Git 提交构建。
   - 构建参数
   - 入口点
   - CMD
   - 用户
   - 工作目录
   - 环境变量
   - 标签
   - 平台
- 漏洞：漏洞部分提供镜像变体的已知 CVE 列表，包括：
   - CVE
   - 严重性
   - 包
   - 修复版本
   - 最后检测时间
   - 状态
   - 已抑制的 CVE
- 证明：变体包括全面的安全证明，以验证镜像的构建过程、内容和安全状况。这些证明已签名，可以使用 cosign 验证。有关可用证明的列表，请参阅 [证明](../core-concepts/attestations.md)。
