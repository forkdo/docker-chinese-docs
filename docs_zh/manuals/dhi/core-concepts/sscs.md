## 什么是软件供应链安全（SSCS）？

软件供应链安全（SSCS）涵盖一系列实践和策略，旨在保护软件开发从初始代码创建到部署和维护的整个生命周期。它专注于保护所有组件，包括代码、依赖项、构建流程和分发渠道，以防止恶意行为者破坏软件供应链。鉴于当前对开源库和第三方组件的依赖日益增加，确保这些元素的完整性和安全性至关重要。

## 为什么 SSCS 很重要？

由于针对软件供应链的复杂网络攻击日益增多，SSCS 的重要性不断上升。近期发生的事件以及开源组件漏洞被利用的情况，凸显了实施强大供应链安全措施的紧迫需求。软件生命周期任何阶段的妥协都可能导致大范围漏洞、数据泄露和重大财务损失。

## Docker Hardened Images 如何助力 SSCS

Docker Hardened Images（DHI）是专为安全而设计的容器镜像，旨在应对现代软件供应链安全的挑战。通过在开发和部署流水线中集成 DHI，您可以借助以下功能增强组织的 SSCS 防护能力：

- **最小攻击面**：DHI 经过精心设计，体积极小，移除了不必要的组件，攻击面减少高达 95%。这种无发行版（distroless）方法最大限度地减少了恶意行为者的潜在入口点。

- **加密签名和来源证明**：每个 DHI 都经过加密签名，确保真实性和完整性。构建来源得到保留，提供镜像来源和构建流程的可验证证据，符合 SLSA（软件制品供应链级别）等标准。

- **软件物料清单（SBOM）**：DHI 包含完整的 SBOM，详细列出镜像中的所有组件和依赖项。这种透明度有助于漏洞管理和合规性追踪，使团队能够有效评估和缓解风险。

- **持续维护和快速 CVE 修复**：Docker 会持续维护 DHI，提供定期更新和安全补丁，并承诺在 SLA 内解决关键和高严重性漏洞。这种主动方法有助于确保镜像始终保持安全，并符合企业标准。
