最小化镜像，有时称为无发行版（distroless）镜像，是移除了不必要的组件（如包管理器、Shell，甚至底层操作系统发行版）的容器镜像。Docker Hardened Images (DHI) 采用这种最小化方法来减少漏洞并强制执行安全的软件交付。[Docker 官方镜像](../../docker-hub/image-library/trusted-content.md#docker-official-images) 和 [Docker 验证发布者镜像](../../docker-hub/image-library/trusted-content.md#verified-publisher-images) 也遵循类似的最小化和安全最佳实践，但可能不会像 DHI 那样彻底精简，以确保与更广泛用例的兼容性。

## 什么是最小化或无发行版镜像？

传统的容器镜像包含完整的操作系统，通常超出运行应用程序所需的范围。相比之下，最小化或无发行版镜像仅包含：

- 应用程序二进制文件
- 其运行时依赖项（例如 libc、Java、Python）
- 任何明确需要的配置或元数据

它们通常排除：

- 操作系统工具（例如 `ls`、`ps`、`cat`）
- Shell（例如 `sh`、`bash`）
- 包管理器（例如 `apt`、`apk`）
- 调试工具（例如 `curl`、`wget`、`strace`）

Docker Hardened Images 基于此模型构建，确保更小且更安全的运行时环境。

## 你获得的优势

| 优势                 | 描述                                                                          |
|----------------------|-------------------------------------------------------------------------------|
| 更小的攻击面         | 更少的组件意味着更少的漏洞和更少的 CVE 暴露风险                                 |
| 更快的启动速度       | 更小的镜像大小带来更快的拉取和启动时间                                          |
| 更高的安全性         | 缺少 Shell 和包管理器限制了攻击者在容器被攻破后可执行的操作                     |
| 更好的合规性         | 更容易进行审计和验证，特别是结合 SBOM 和证明（attestations）时                  |

## 应对常见的权衡

最小化和无发行版镜像提供了强大的安全优势，但它们可能会改变你使用容器的方式。Docker Hardened Images 旨在在提升安全性的同时保持生产力。

| 关注点       | Docker Hardened Images 如何帮助                                                                                                                                                                                         |
|--------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 可调试性     | Hardened 镜像默认排除 Shell 和 CLI 工具。使用 [Docker Debug](../../../reference/cli/docker/debug.md) 可以临时附加一个调试辅助容器进行故障排除，而无需修改原始容器。                                                   |
| 熟悉度       | DHI 支持多种基础镜像，包括 Alpine 和 Debian 变体，因此你可以选择熟悉的环境，同时仍然受益于强化实践。                                                                                                                    |
| 灵活性       | 运行时不可变性有助于保护容器安全。使用多阶段构建和 CI/CD 来控制变更，并在开发过程中可选地使用面向开发的基础镜像。                                                                                                      |

通过在最小化和实用工具之间取得平衡，Docker Hardened Images 支持现代开发工作流，同时不牺牲安全性和可靠性。

## 使用最小化镜像的最佳实践

- 使用多阶段构建来分离构建时和运行时环境
- 使用 CI 流水线验证镜像行为，而不是交互式检查
- 在 Dockerfile 中显式包含运行时特定的依赖项
- 使用 Docker Scout 持续监控 CVE，即使在最小化镜像中也是如此

通过 Docker Hardened Images 采用最小化或无发行版镜像，你可以获得一个更安全、更可预测、更适合生产的容器环境，专为自动化、清晰性和降低风险而设计。
