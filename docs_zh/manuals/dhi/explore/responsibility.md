Docker Hardened Images (DHIs) 由 Docker 精心策划和维护，基于上游开源组件构建。为了提供安全性、可靠性和合规性，职责在以下三个群体之间共享：

- 上游维护者：负责包含在 DHI 中的开源软件的开发者和社区。
- Docker：提供加固、签名并维护的容器镜像。
- 您（客户）：在您的环境中运行并可选地自定义 DHIs 的使用者。

本主题概述了各方的职责，以便您能够有效且安全地使用 DHIs。

## 发布

- 上游：发布并维护 DHI 中包含的软件组件的官方版本。包括版本控制、变更日志和弃用通知。
- Docker：基于上游版本构建、加固并签名 Docker Hardened Images。Docker 根据上游发布时间线和内部策略维护这些镜像。
- 您：确保您使用的 DHI 和上游项目版本处于受支持状态。使用过时或不受支持的组件可能引入安全风险。

## 打补丁

- 上游：维护每个组件的源代码，包括修复库和依赖项中的漏洞。
- Docker：应用上游补丁后重建并重新发布镜像。Docker 监控漏洞并在受影响的镜像中发布更新。只有 DHI Enterprise 包含 SLA。DHI Free 提供安全基线，但无保证的修复时间表。
- 您：在您的环境中应用 DHI 更新，并修补您在基础镜像之上安装的任何软件或依赖项。

## 测试

- 上游：定义原始软件的行为和功能，并负责验证核心特性。
- Docker：验证 DHI 能够启动、运行并与上游期望保持一致。Docker 还运行安全扫描，并在每个镜像中包含[测试证明](../core-concepts/attestations.md)。
- 您：在 DHI 之上测试您的应用程序，并验证您的任何更改或自定义在您的环境中按预期运行。

## 安全与合规

- Docker：为每个镜像发布已签名的 SBOM、VEX 文档、来源数据和 CVE 扫描结果，以支持合规性和供应链安全。
  - 对于免费 DHI 用户：所有安全元数据和透明度功能均免费提供。
  - 对于 DHI Enterprise 用户：提供额外的合规变体（如 FIPS 和 STIG）和自定义功能，当基础镜像打补丁时会自动重建。
- 您：将 DHI 集成到您的安全和合规工作流程中，包括漏洞管理和审计。

## 支持

- Docker：
  - 对于免费 DHI 用户：提供社区支持和公开文档。
  - 对于 DHI Enterprise 用户：可访问 Docker 的企业支持团队，用于关键任务应用程序。
- 您：监控 Docker 的发布说明、安全公告和文档，了解更新和最佳实践。

## 总结

Docker Hardened Images 为您提供了一个安全的基础，包含已签名的元数据和上游透明度。您的职责是明智地使用这些镜像，及时应用更新，并验证您的配置和应用程序满足您的内部要求。
