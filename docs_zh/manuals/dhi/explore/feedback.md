为确保 Docker Hardened Images (DHI) 的质量、安全性和可靠性，我们创建了一个仓库作为联系点，鼓励社区协作改进 Hardened Images 生态系统。

## 问题或讨论

你可以使用 [GitHub Discussions
论坛](https://github.com/orgs/docker-hardened-images/discussions) 与 DHI 团队互动，包括：

- 关于 DHI 的一般性问题
- 最佳实践和建议
- 安全技巧和建议
- 展示和分享你的实现
- 社区公告

## 报告错误或问题

你可以 [创建新问题](https://github.com/docker-hardened-images/catalog/issues) 来讨论以下主题：

- 错误报告
- 功能请求
- 文档改进
- 安全漏洞（参见安全策略）

建议先搜索现有问题，确认该问题是否已被报告。DHI 团队会定期审查报告，并感谢清晰、可操作的反馈。

## 负责任的安全披露

禁止在协调披露和解决之前发布漏洞的详细信息。

如果你发现安全漏洞，请遵循 Docker 的 [安全披露政策](https://www.docker.com/trust/vulnerability-disclosure-policy/) 进行负责任的报告。
