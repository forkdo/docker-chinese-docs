为了维护 Docker Hardened Images (DHI) 的质量、安全性和可靠性，
我们创建了一个仓库作为联系点，鼓励社区协作改进 Hardened Images 生态系统。

## 问题或讨论

您可以在 [GitHub Discussions
论坛](https://github.com/orgs/docker-hardened-images/discussions) 上与 DHI 团队互动，讨论以下话题：

- 关于 DHIs 的一般性问题
- 最佳实践和建议
- 安全技巧和建议
- 展示和分享您的实现
- 社区公告

## 报告 bug 或问题

您可以通过 [创建新问题](https://github.com/docker-hardened-images/catalog/issues) 来报告以下主题：

- Bug 报告
- 功能请求
- 文档改进
- 安全漏洞（参见安全策略）

建议您先搜索现有问题，查看是否已被报告。
DHI 团队会定期审查报告，并感谢清晰、可操作的反馈。

## 负责任的安全披露

在协调披露和解决之前，禁止发布漏洞的详细信息。

如果您发现安全漏洞，请遵循 Docker 的 [安全披露](https://www.docker.com/trust/vulnerability-disclosure-policy/) 政策，负责任地报告。
