## 17.04.0-ce
2017-04-05

### 构建器

* 禁用构建容器的日志功能 [#29552](https://github.com/docker/docker/pull/29552)
* 修复 `.dockerignore` 中 `**/` 的使用问题 [#29043](https://github.com/docker/docker/pull/29043)

### 客户端

+ 对 `docker stack ls` 按名称排序 [#31085](https://github.com/docker/docker/pull/31085)
+ 增加指定绑定挂载一致性的标志 [#31047](https://github.com/docker/docker/pull/31047)
* Docker CLI --help 的输出现在会根据终端宽度自动换行 [#28751](https://github.com/docker/docker/pull/28751)
* 在 docker ps 中隐藏镜像摘要 [#30848](https://github.com/docker/docker/pull/30848)
* 隐藏与 Windows 相关的命令选项 [#30788](https://github.com/docker/docker/pull/30788)
* 修复 `docker plugin install` 提示，允许按 "enter" 键选择默认的 "N" [#30769](https://github.com/docker/docker/pull/30769)
+ 为 Go 模板添加 `truncate` 函数 [#30484](https://github.com/docker/docker/pull/30484)
* 在 `stack deploy` 中支持扩展语法的端口配置 [#30476](https://github.com/docker/docker/pull/30476)
* 在 `stack deploy` 中支持扩展语法的挂载配置 [#30597](https://github.com/docker/docker/pull/30597) [#31795](https://github.com/docker/docker/pull/31795)
+ 为 docker build 添加 `--add-host` 选项 [#30383](https://github.com/docker/docker/pull/30383)
+ 为 `docker network ls --format` 添加 `.CreatedAt` 占位符 [#29900](https://github.com/docker/docker/pull/29900)
* 更新 `--secret-rm` 和 `--secret-add` 的顺序 [#29802](https://github.com/docker/docker/pull/29802)
+ 为 `docker plugin ls` 添加 `--filter enabled=true` 选项 [#28627](https://github.com/docker/docker/pull/28627)
+ 为 `docker service ls` 添加 `--format` 选项 [#28199](https://github.com/docker/docker/pull/28199)
+ 为 `docker ps --filter` 添加 `publish` 和 `expose` 过滤器 [#27557](https://github.com/docker/docker/pull/27557)
* 在 `docker service ps` 中支持多个服务 ID [#25234](https://github.com/docker/docker/pull/25234)
+ 允许使用 `--availability=drain` 加入 swarm [#24993](https://github.com/docker/docker/pull/24993)
* 当启用 AppArmor 且未定义其他配置文件时，docker inspect 现在会显示 "docker-default" [#27083](https://github.com/docker/docker/pull/27083)

### 日志

+ 为容器日志实现可选的环形缓冲区 [#28762](https://github.com/docker/docker/pull/28762)
+ 为 awslogs (CloudWatch) 添加 `--log-opt awslogs-create-group=<true|false>` 选项，以支持按需创建日志组 [#29504](https://github.com/docker/docker/pull/29504)
- 修复在使用 "静态" 二进制文件时 gcplogs 日志驱动程序导致的段错误 [#29478](https://github.com/docker/docker/pull/29478)


### 网络

* 检查 `docker network connect` 中的 `--ip`、`--ip6` 和 `--link-local-ip` 参数 [#30807](https://github.com/docker/docker/pull/30807)
+ 添加对 `dns-search` 的支持 [#30117](https://github.com/docker/docker/pull/30117)
+ 为 docker network inspect 添加 --verbose 选项，以从所有 swarm 节点显示任务详细信息 [#31710](https://github.com/docker/docker/pull/31710)
* 加入集群时清除过时的数据路径加密状态 [docker/libnetwork#1354](https://github.com/docker/libnetwork/pull/1354)
+ 确保 iptables 初始化只执行一次 [docker/libnetwork#1676](https://github.com/docker/libnetwork/pull/1676)
* 修复 iptables filter 规则的错误顺序 [docker/libnetwork#961](https://github.com/docker/libnetwork/pull/961)
+ 在可附加网络上为服务记录添加匿名容器别名 [docker/libnetwork#1651](https://github.com/docker/libnetwork/pull/1651)
+ 支持 `com.docker.network.container_iface_prefix` 驱动标签 [docker/libnetwork#1667](https://github.com/docker/libnetwork/pull/1667)
+ 通过省略未使用的网络详细信息来提高网络列表性能 [#30673](https://github.com/docker/docker/pull/30673)

### 运行时

* 在未设置 live-restore 时恢复暂停的容器 [#31704](https://github.com/docker/docker/pull/31704)
- 不允许在 Dockerfile 的健康检查选项中使用秒以下的单位 [#31177](https://github.com/docker/docker/pull/31177)
* 在 `secret update` 中支持名称和 ID 前缀 [#30856](https://github.com/docker/docker/pull/30856)
* 在 websocket attach 端点使用二进制帧 [#30460](https://github.com/docker/docker/pull/30460)
* 修复 Linux 挂载调用不应用传播类型更改的问题 [#30416](https://github.com/docker/docker/pull/30416)
* 修复 `exec -i` 失败时的 ExecIds 泄漏问题 [#30340](https://github.com/docker/docker/pull/30340)
* 如果 `danglingOnly=true`，则清理已命名但未打标签的镜像 [#30330](https://github.com/docker/docker/pull/30330)
+ 添加守护进程标志以将 `no_new_priv` 设置为非特权容器的默认值 [#29984](https://github.com/docker/docker/pull/29984)
+ 添加守护进程选项 `--default-shm-size` [#29692](https://github.com/docker/docker/pull/29692)
+ 支持注册表镜像配置重新加载 [#29650](https://github.com/docker/docker/pull/29650)
- 构建镜像时忽略守护进程日志配置 [#29552](https://github.com/docker/docker/pull/29552)
* 将 secret 名称或 ID 前缀解析从客户端移到守护进程 [#29218](https://github.com/docker/docker/pull/29218)
+ 允许在容器创建/运行时向 `cgroup devices.allow` 添加规则 [#22563](https://github.com/docker/docker/pull/22563)
- 修复运行 `systemd daemon-reload` 时 `cpu.cfs_quota_us` 被重置的问题 [#31736](https://github.com/docker/docker/pull/31736)
