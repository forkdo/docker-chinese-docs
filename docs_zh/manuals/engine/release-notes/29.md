本文档描述了 Docker Engine v29 的最新变更、新增功能、已知问题和修复。

如需了解：

- 已弃用和移除的功能，请参阅 [已弃用的 Engine 功能](../deprecated.md)。

## 29.1.3

{{< release-date date="2025-12-12" >}}

本次发布的完整 Pull Request 和变更列表，请参考以下 GitHub milestones：

- [docker/cli, 29.1.3 milestone](https://github.com/docker/cli/issues?q=is%3Aclosed+milestone%3A29.1.3)
- [moby/moby, 29.1.3 milestone](https://github.com/moby/moby/issues?q=is%3Aclosed+milestone%3A29.1.3)

### 错误修复和增强

- 为 `docker stack deploy --compose-file` 添加 shell 补全。[docker/cli#6690](https://github.com/docker/cli/pull/6690)
- containerd 镜像存储：修复导致 `docker build` 忽略显式设置的 `unpack` 镜像导出器选项的 bug。[moby/moby#51514](https://github.com/moby/moby/pull/51514)
- 修复 `docker image ls` 对悬空镜像的处理。[docker/cli#6704](https://github.com/docker/cli/pull/6704)
- 修复一个 bug，该 bug 可能导致引擎在关闭时将设置了 autoremove 的容器留在 'dead' 状态，且永不回收。[moby/moby#51693](https://github.com/moby/moby/pull/51693)
- 修复 i386 架构上的构建。[moby/moby#51528](https://github.com/moby/moby/pull/51528)
- 修复显式配置 graphdriver（`"storage-driver"`）在存在旧 graphdriver 状态时被当作 containerd 快照器的问题。[moby/moby#51516](https://github.com/moby/moby/pull/51516)
- 修复可能导致创建孤儿 overlay2 层的 bug。[moby/moby#51703](https://github.com/moby/moby/pull/51703)

### 网络

- 允许在未为网络配置特定子网时，创建具有特定 IP 地址的容器。[moby/moby#51583](https://github.com/moby/moby/pull/51583)
- 修复在 v29.1.2 升级前通过 API 创建、使用 `PublishAll` 且 `PortBindings` 映射为 nil 的容器启动时崩溃的问题。[moby/moby#51691](https://github.com/moby/moby/pull/51691)
- 修复一个 bug，该 bug 导致节点加入 Swarm 集群后，无法解析连接到非 Swarm 范围网络的容器的 DNS。[moby/moby#51515](https://github.com/moby/moby/pull/51515)
- 修复使用远程网络驱动插件时导致守护进程崩溃的问题。[moby/moby#51558](https://github.com/moby/moby/pull/51558)
- 修复一个可能导致创建多网络连接容器时出现 "endpoint not found" 错误的问题，当其中一个网络是非内部网络但没有自己的外部 IP 连接时。[moby/moby#51538](https://github.com/moby/moby/pull/51538)
- 修复根用户无权 Docker 在禁用 IPv6 的主机上启动的问题。[moby/moby#51543](https://github.com/moby/moby/pull/51543)
- 当容器创建时端口映射指向容器端口 0 时返回错误。[moby/moby#51695](https://github.com/moby/moby/pull/51695)

## 29.1.2

{{< release-date date="2025-12-02" >}}

本次发布的完整 Pull Request 和变更列表，请参考以下 GitHub milestones：

- [docker/cli, 29.1.2 milestone](https://github.com/docker/cli/issues?q=is%3Aclosed+milestone%3A29.1.2)
- [moby/moby, 29.1.2 milestone](https://github.com/moby/moby/issues?q=is%3Aclosed+milestone%3A29.1.2)

### 安全

- 更新 Go 运行为 [1.25.5](https://go.dev/doc/devel/release#go1.25.5)。[moby/moby#51648](https://github.com/moby/moby/pull/51648)，[docker/cli#6688](https://github.com/docker/cli/pull/6688)
  - 修复了格式化主机名验证错误时过度资源使用导致的潜在 DoS 问题 [**CVE-2025-61729**](https://nvd.nist.gov/vuln/detail/CVE-2025-61729)
  - 修复了通配符 SAN 的子域约束错误执行问题，该问题可能导致错误信任证书 [**CVE-2025-61727**](https://nvd.nist.gov/vuln/detail/CVE-2025-22874)

### 错误修复和增强

- containerd 镜像存储：修复 `docker image inspect` 在本地并非所有可分发 blob 都可用时无法返回可用镜像数据的问题。[moby/moby#51629](https://github.com/moby/moby/pull/51629)
- dockerd-rootless-setuptool.sh：修复 `nsenter: no namespace specified`。[moby/moby#51622](https://github.com/moby/moby/pull/51622)
- 修复使用图驱动作为存储时，`docker system df` 显示共享大小和唯一大小为 `N/A` 的问题。[moby/moby#51631](https://github.com/moby/moby/pull/51631)

### 打包更新

- 更新 runc（静态二进制文件中）到 [v1.3.4](https://github.com/opencontainers/runc/releases/tag/v1.3.4)。[moby/moby#51633](https://github.com/moby/moby/pull/51633)

### 网络

- 修复在使用 slirp4netns 时，根用户无权模式下端口映射失败的 bug。[moby/moby#51616](https://github.com/moby/moby/pull/51616)
- 防止在 API 请求中设置 `HostConfig.PublishAllPorts`（`-P`）且无端口绑定时崩溃。[moby/moby#51621](https://github.com/moby/moby/pull/51621)

## 29.0.0

{{< release-date date="2025-11-10" >}}

本次发布的完整 Pull Request 和变更列表，请参考以下 GitHub milestones：

- [docker/cli, 29.0.0 milestone](https://github.com/docker/cli/issues?q=is%3Aclosed+milestone%3A29.0.0)
- [moby/moby, 29.0.0 milestone](https://github.com/moby/moby/issues?q=is%3Aclosed+milestone%3A29.0.0)

> [!CAUTION]
> 本次发布包含多个破坏性变更和弃用。升级前请仔细查看发布说明。

- 现在可以通过将 Docker 守护进程的 `firewall-backend` 选项设置为 `nftables` 来启用 nftables 的实验性支持。更多信息请参阅 [Docker Engine 文档](https://docs.docker.com/engine/network/firewall-nftables/)。
- 对于**全新安装**，containerd 镜像存储现在是默认选项。这不适用于配置了 `userns-remap` 的守护进程（参见 [moby#47377](https://github.com/moby/moby/issues/47377)）。
