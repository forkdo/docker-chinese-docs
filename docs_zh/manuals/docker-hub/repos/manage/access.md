在本主题中，了解可用于管理仓库访问权限的功能。包括可见性、协作者、角色、团队和组织访问令牌。

## 仓库可见性

最基本的仓库访问通过可见性控制。仓库的可见性可以是公开的或私有的。

在公开可见性下，仓库会出现在 Docker Hub 搜索结果中，并且每个人都可以拉取。要管理公开个人仓库的推送访问，可以使用协作者。要管理公开组织仓库的推送访问，可以使用角色、团队或组织访问令牌。

在私有可见性下，仓库不会出现在 Docker Hub 搜索结果中，仅对被授予权限的用户可见。要管理私有个人仓库的推送和拉取访问，可以使用协作者。要管理私有组织仓库的推送和拉取访问，可以使用角色、团队或组织访问令牌。

### 更改仓库可见性

在 Docker Hub 中创建仓库时，可以设置仓库可见性。此外，您可以在个人仓库设置中设置创建仓库时的默认可见性。以下描述如何在仓库创建后更改可见性。

要更改仓库可见性：

1. 登录到 [Docker Hub](https://hub.docker.com)。
2. 选择 **My Hub** > **Repositories**。
3. 选择一个仓库。

   仓库的 **General** 页面出现。

4. 选择 **Settings** 选项卡。
5. 在 **Visibility settings** 下，选择以下之一：

   - **Make public**：仓库出现在 Docker Hub 搜索结果中，每个人都可以拉取。
