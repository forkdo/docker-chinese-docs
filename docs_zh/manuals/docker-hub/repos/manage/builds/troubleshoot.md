## 构建失败

如果构建失败，**常规**和**构建**选项卡上的构建报告行旁边会出现一个**重试**图标。**构建报告**页面和**时间线日志**也会显示一个**重试**按钮。

![显示重试构建按钮的时间线视图](images/retry-build.png)

> [!NOTE]
>
> 如果您正在查看属于某个组织的仓库的构建详情，只有当您对该仓库拥有 `读写` 权限时，才会显示**取消**和**重试**按钮。

自动构建有 4 小时的执行时间限制。如果构建达到此时间限制，它将自动取消，构建日志会显示以下消息：

```text
2022-11-02T17:42:27Z The build was cancelled or exceeded the maximum execution time.
```

此日志消息与您主动取消构建时相同。要确定构建是否被自动取消，请检查构建持续时间。

## 构建带有链接的私有子模块的仓库

Docker Hub 会在您的源代码仓库中设置一个部署密钥，允许其克隆仓库并进行构建。此密钥仅适用于单个特定的代码仓库。如果您的源代码仓库使用私有 Git 子模块，或者需要克隆其他私有仓库来构建，Docker Hub 将无法访问这些额外的仓库，您的构建将无法完成，并且错误会记录在您的构建时间线中。

要解决此问题，您可以设置自动构建，使用 `SSH_PRIVATE` 环境变量覆盖部署密钥，并授予 Docker Hub 的构建系统访问仓库的权限。

> [!NOTE]
>
> 如果您正在为团队使用自动构建，请改用以下流程，并为您的源代码提供商配置一个服务用户。您也可以为个人帐户执行此操作，以限制 Docker Hub 对源仓库的访问。

1.  生成一个仅用于构建的 SSH 密钥对，并将公钥添加到您的源代码提供商帐户。

    此步骤是可选的，但允许您在不移除其他访问权限的情况下撤销仅用于构建的密钥对。

2.  将密钥对的私有部分复制到剪贴板。
3.  在 Docker Hub 中，导航到具有链接的私有子模块的仓库的构建页面。（如有必要，请[按照此处的步骤](index.md#configure-automated-builds)配置自动构建。）
4.  在屏幕底部，选择**构建环境变量**旁边的**加号**图标。
5.  输入 `SSH_PRIVATE` 作为新环境变量的名称。
6.  将密钥对的私有部分粘贴到**值**字段中。
7.  选择**保存**或**保存并构建**以验证构建现在是否完成。

> [!NOTE]
>
> 您必须使用 SSH 协议的 git clone (`git@submodule.tld:some-submodule.git`) 而不是 HTTPS 来配置您的私有 git 子模块。
