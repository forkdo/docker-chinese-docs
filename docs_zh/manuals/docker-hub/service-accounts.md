> [!IMPORTANT]
>
> 自 2024 年 12 月 10 日起，增强型服务账号附加组件已不再提供。
> 现有的服务账号协议将在当前期限内继续有效，但不再接受新的购买或续订增强型服务账号附加组件，客户必须在新的订阅下续订。
>
> Docker 建议迁移到 [组织访问令牌 (OATs)](/manuals/enterprise/security/access-tokens.md)，它可以提供类似的功能。

服务账号是用于自动管理容器镜像或容器化应用程序的 Docker ID。服务账号通常用于自动化工作流中，不与组织中的成员共享 Docker ID。服务账号的常见使用场景包括在 Docker Hub 上镜像内容，或在 CI/CD 流程中集成镜像拉取操作。

## 增强型服务账号附加组件等级

以下表格详细说明了增强型服务账号附加组件的等级：

| 等级 | 每日拉取次数\* |
| ------ | ------ |
| 1 | 5,000-10,000 |
| 2 | 10,000-25,000 |
| 3 | 25,000-50,000 |
| 4 | 50,000-100,000 |
| 5 | 100,000+ |

<sub>*服务账号在一年内最多可超出拉取次数 25%，最多持续 20 天，不会产生额外费用。消费报告可根据请求提供。<sub>
