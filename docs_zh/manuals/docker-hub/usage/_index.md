下表概述了每种用户类型的包含使用情况和限制，受公平使用条款约束：

| 用户类型                | 每 6 小时拉取限制             | 公共仓库数量 | 私有仓库数量 |
|--------------------------|-----------------------------------------|-------------------------------|--------------------------------|
| Business（已认证） | 无限制                               | 无限制                     | 无限制                      |
| Team（已认证）     | 无限制                               | 无限制                     | 无限制                      |
| Pro（已认证）      | 无限制                               | 无限制                     | 无限制                      |
| Personal（已认证）      | 200                                     | 无限制                     | 最多 1 个                        |
| 未认证用户    | 每个 IPv4 地址或 IPv6 /64 子网 100 次 | 不适用                | 不适用                 |

更多详细信息，请参阅 [拉取使用情况和限制](./pulls.md)。

## 公平使用

使用 Docker 平台时，用户应注意，过度的数据传输、拉取速率或数据存储可能导致限流或额外收费。为确保资源的公平使用并维持服务质量，我们保留对表现出过度数据和存储消耗的账户施加限制或收取额外费用的权利。

### 滥用率限制

Docker Hub 设有滥用率限制，以保护应用程序和基础设施。此限制适用于对 Hub 属性的所有请求，包括网页、API 和镜像拉取。限制按每个 IPv4 地址或每个 IPv6 /64 子网应用，虽然限制会根据负载等因素随时间变化，但大致在每分钟数千次请求的范围内。滥用限制对所有用户一视同仁，无论账户级别如何。

您可以通过查看错误代码来区分拉取率限制和滥用率限制。滥用限制返回简单的 `429 Too Many Requests` 响应。拉取限制返回更长的错误消息，其中包含指向文档的链接。
