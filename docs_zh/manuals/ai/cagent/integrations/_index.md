cagent 代理可以集成到不同的环境中，具体取决于您的使用需求。每种集成类型都有其特定用途。

## 集成类型

### ACP - 编辑器集成

直接在您的编辑器（Neovim、Zed）中运行 cagent 代理。代理可以看到编辑器的文件上下文，并通过编辑器界面读取和修改文件。当您希望在编辑器中嵌入 AI 编程助手时，请使用 ACP。

设置说明请参阅 [ACP 集成](./acp.md)。

### MCP - 工具集成

将 cagent 代理作为工具暴露给 MCP 客户端（如 Claude Desktop 或 Claude Code）。您的代理会出现在客户端的工具列表中，客户端可以在需要时调用它们。当您希望 Claude Desktop（或其他 MCP 客户端）能够访问您的专业代理时，请使用 MCP。

设置说明请参阅 [MCP 集成](./mcp.md)。

### A2A - 代理间通信

将 cagent 代理作为 HTTP 服务器运行，其他代理或系统可以使用代理间（Agent-to-Agent）协议调用它们。您的代理会成为一个服务，其他系统可以通过网络发现并调用它。当您要构建多代理系统或将您的代理作为 HTTP 服务暴露时，请使用 A2A。

设置说明请参阅 [A2A 集成](./a2a.md)。

## 选择合适的集成方式

| 功能         | ACP                | MCP                | A2A                  |
| ------------ | ------------------ | ------------------ | -------------------- |
| 用例         | 编辑器集成         | 代理作为工具       | 代理间调用           |
| 传输协议     | stdio              | stdio/SSE          | HTTP                 |
| 发现方式     | 编辑器插件         | 服务器清单         | 代理卡片             |
| 最适合       | 代码编辑           | 工具集成           | 多代理系统           |
| 通信方式     | 编辑器调用代理     | 客户端调用工具     | 代理间通信           |

如果您希望代理嵌入在编辑器中辅助编码，请选择 ACP。如果您希望 Claude Desktop（或其他 MCP 客户端）能够将您的专业代理作为工具调用，请选择 MCP。如果您正在构建多代理系统，其中代理需要通过 HTTP 相互通信，请选择 A2A。
