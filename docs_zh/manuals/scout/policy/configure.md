某些策略类型是可配置的。这意味着你可以基于该策略类型创建具有自定义配置参数的新版本策略。如果需要临时忽略某个策略，也可以禁用它；如果策略不再符合你的需求，也可以将其完全删除。

> [!NOTE]
> 如果你删除或自定义了策略，该策略默认配置的历史评估结果将被移除。

## 添加策略

要添加新策略，首先选择你想要自定义的策略类型。所有自定义策略都以策略类型为基础。

你可以编辑新策略的显示名称和描述，以便更好地传达策略的合规和不合规状态。但你无法更改策略类型的名称，只能更改其显示名称。

策略的可用配置参数取决于你正在编辑的策略类型。更多信息，请参考 [策略类型](/manuals/scout/policy/_index.md#policy-types)。

添加策略的步骤：

1. 在 Docker Scout 仪表板中，进入 [策略页面](https://scout.docker.com/reports/policy)。
2. 选择 **添加策略** 按钮，打开策略配置界面。
3. 在策略配置界面中，找到你想要配置的策略类型，选择 **配置** 以打开策略配置页面。

   - 如果 **配置** 按钮为灰色，表示当前策略没有可配置的参数。
   - 如果按钮显示为 **集成**，表示需要先完成设置才能启用该策略。选择 **集成** 将引导你进入该集成的设置指南。

4. 更新策略参数。
5. 保存更改：

   - 选择 **保存策略** 以提交更改并为当前组织启用策略。
   - 选择 **保存并禁用** 以保存策略配置但不启用。

## 编辑策略

编辑策略允许你修改其配置，而无需从头创建新策略。当你需要根据需求变化或组织合规目标的调整来修改策略参数时，这非常有用。

编辑策略的步骤：

1. 在 Docker Scout 仪表板中，进入 [策略页面](https://scout.docker.com/reports/policy)。
2. 选择你想要编辑的策略。
3. 选择 **编辑** 按钮。
4. 更新策略参数。
5. 保存更改。

## 禁用策略

禁用策略后，该策略的评估结果将被隐藏，不再在 Docker Scout 仪表板或 CLI 中显示。禁用策略不会删除历史评估结果，因此如果你改变主意稍后重新启用策略，早期评估的结果仍然可用。

禁用策略的步骤：

1. 在 Docker Scout 仪表板中，进入 [策略页面](https://scout.docker.com/reports/policy)。
2. 选择你想要禁用的策略。
3. 选择 **禁用** 按钮。

## 删除策略

删除策略后，该策略的评估结果也会被删除，不再在 Docker Scout 仪表板或 CLI 中显示。

删除策略的步骤：

1. 在 Docker Scout 仪表板中，进入 [策略页面](https://scout.docker.com/reports/policy)。
2. 选择你想要删除的策略。
3. 选择 **删除** 按钮。

## 恢复已删除的策略

如果你删除了策略，可以通过遵循 [添加策略](#add-a-policy) 中的步骤重新创建。在策略配置界面中，选择你想要恢复的已删除策略上的 **配置**。
