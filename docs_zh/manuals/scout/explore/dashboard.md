[Docker Scout Dashboard](https://scout.docker.com/) 帮助您与团队共享组织中镜像的分析结果。开发者现在可以一览他们在 Docker Hub 上所有镜像的安全状态概览，并即时获取修复建议。它也帮助安全、合规和运维等角色的团队成员了解需要重点关注的漏洞和问题。

## 概述

![Docker Scout Dashboard 概览截图](../images/dashboard-overview.webp?border=true)

**概览** 选项卡提供了所选组织中仓库的摘要信息。

在页面顶部，您可以选择要查看的 **环境**。默认情况下，显示最近推送的镜像。如需了解环境的更多信息，请参阅 [环境监控](/manuals/scout/integrations/environment/_index.md)。

**策略** 框显示您当前每个策略的合规评级，以及所选环境的趋势指示。趋势描述了最新镜像相比前一版本的策略变化。有关策略的更多信息，请参阅 [策略评估](/manuals/scout/policy/_index.md)。

漏洞图表显示了所选环境中镜像的漏洞总数随时间的变化。您可以使用下拉菜单配置图表的时间尺度。

使用网站顶部的标题菜单访问 Docker Scout Dashboard 的不同主要部分：

- **策略**：显示组织的策略合规情况，参见 [策略](#策略)
- **镜像**：列出组织中所有启用 Docker Scout 的仓库，参见 [镜像](#镜像)
- **基础镜像**：列出组织中仓库使用的所有基础镜像
- **软件包**：列出组织中仓库的所有软件包
- **漏洞**：列出组织镜像中的所有 CVE，参见 [漏洞](#漏洞)
- **集成**：创建和管理第三方集成，参见 [集成](#集成)
- **设置**：管理仓库设置，参见 [设置](#设置)

## 策略

**策略** 视图显示所选组织和环境中所有镜像的策略合规情况细分。您可以使用 **镜像** 下拉菜单查看特定环境的策略细分。

有关策略的更多信息，请参阅 [策略评估](/manuals/scout/policy/_index.md)。

## 镜像

**镜像** 视图显示所选环境中 Scout 启用仓库的所有镜像。您可以选择不同的环境或使用文本过滤器按仓库名称过滤列表。

![镜像视图截图](../images/dashboard-images.webp)

对于每个仓库，列表显示以下详细信息：

- 仓库名称（不含标签或摘要的镜像引用）
- 所选环境中镜像的最新标签
- 最新标签的操作系统和架构
- 最新标签的漏洞状态
- 最新标签的策略状态

选择仓库链接将带您到该仓库中所有已分析镜像的列表。从这里您可以查看特定镜像的完整分析结果，并比较标签以查看软件包和漏洞的差异。

选择镜像链接将带您到所选标签或摘要的详细信息视图。此视图包含两个选项卡，详细说明镜像的组成和策略合规情况：

- **策略状态** 显示所选镜像的策略评估结果。这里您还可以获得有关策略违规详情的链接。

  有关策略的更多信息，请参阅 [策略评估](/manuals/scout/policy/_index.md)。

- **镜像层** 显示镜像分析结果的细分。您可以完整了解镜像包含的漏洞，并理解它们是如何进入的。

## 漏洞

**漏洞** 视图显示组织中镜像的所有漏洞列表。此列表包括有关 CVE 的详细信息，如严重性和通用漏洞评分系统 (CVSS) 分数，以及是否有可用的修复版本。此处显示的 CVSS 分数是所有可用 [来源](/manuals/scout/deep-dive/advisory-db-sources.md) 中的最高分数。

选择此页面上的链接将打开漏洞详情页面，这是一个公开可见的页面，显示 CVE 的详细信息。即使其他人不是您的 Docker 组织成员或未登录 Docker Scout，您也可以与他们分享特定 CVE 描述的链接。

如果您已登录，此页面上的 **我的镜像** 选项卡将列出受该 CVE 影响的所有您的镜像。

## 集成

**集成** 页面允许您创建和管理 Docker Scout 集成，例如环境集成和注册表集成。有关如何开始使用集成的更多信息，请参阅 [将 Docker Scout 与其他系统集成](/manuals/scout/integrations/_index.md)。

## 设置

Docker Scout Dashboard 中的设置菜单包含：

- [**仓库设置**](#仓库设置) 用于启用和禁用仓库
- [**通知**](#通知设置) 用于管理您的 Docker Scout 通知偏好。

### 仓库设置

启用仓库的 Docker Scout 后，Docker Scout 会在您推送到该仓库时自动分析新标签。要在 Amazon ECR、Azure ACR 或其他第三方注册表中启用仓库，您首先需要集成它们。参见 [容器注册表集成](/manuals/scout/integrations/_index.md#container-registries)

### 通知设置

[通知设置](https://scout.docker.com/settings/notifications) 页面是您更改 Docker Scout 通知偏好的地方。通知设置是个人的，更改通知设置只影响您的个人账户，不影响整个组织。

Docker Scout 中通知的目的是提醒您注意影响您的上游变化。Docker Scout 会在安全公告中披露新漏洞并影响您一个或多个镜像时通知您。由于推送新镜像导致的漏洞暴露或策略合规性变化不会触发通知。

> [!NOTE]
>
> 通知仅针对每个仓库的 *最后推送* 镜像标签触发。"最后推送" 指的是最近推送到注册表并由 Docker Scout 分析的镜像标签。如果最后推送的镜像不受新披露的 CVE 影响，则不会触发通知。

可用的通知设置包括：

- **仓库范围**

  在这里您可以选择是否要为所有仓库启用通知，或仅为特定仓库启用。这些设置适用于当前选择的组织，可以为您所属的每个组织单独更改。

  - **所有仓库**：选择此选项可接收所有您有访问权限的仓库的通知。
  - **特定仓库**：选择此选项可接收特定仓库的通知。然后您可以输入要接收通知的仓库名称。

- **交付偏好**

  这些设置控制您如何接收 Docker Scout 的通知。它们适用于您所属的所有组织。

  - **通知弹窗**：选择此复选框可在 Docker Scout Dashboard 中接收通知弹窗消息。
  - **操作系统通知**：选择此复选框，如果 Docker Scout Dashboard 在浏览器标签页中打开，您将从浏览器接收操作系统级通知。

要启用操作系统通知，Docker Scout 需要使用浏览器 API 发送通知的权限。

从该页面，您还可以前往团队协作集成的设置，例如 [Slack](/manuals/scout/integrations/team-collaboration/slack.md) 集成。

您也可以在 Docker Desktop 中通过 **设置** > **通知** 配置您的通知设置。
