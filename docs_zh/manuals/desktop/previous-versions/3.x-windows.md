## Docker Desktop 3.6.0
2021-08-11

### 新增功能

- **开发环境**：您现在可以从本地 Git 仓库创建开发环境。
- **卷管理**：您现在可以按名称、创建日期和卷大小对卷进行排序。还可以使用**搜索**字段搜索特定卷。更多信息请参阅[浏览卷](../use-desktop/volumes.md)。

### 升级组件

- [Compose V2 RC1](https://github.com/docker/compose-cli/releases/tag/v2.0.0-rc.1)
  - Docker compose 命令行自动补全。
  - 允许设置 0 个副本/扩展数。
  - 在日志中检测新容器时支持 --follow 选项。
- [Go 1.16.7](https://github.com/golang/go/releases/tag/go1.16.7)
- [Docker Engine 20.10.8](/manuals/engine/release-notes/20.10.md#20108)
- [containerd v1.4.9](https://github.com/containerd/containerd/releases/tag/v1.4.9)
- [runc v1.0.1](https://github.com/opencontainers/runc/releases/tag/v1.0.1)
- [Kubernetes 1.21.3](https://github.com/kubernetes/kubernetes/releases/tag/v1.21.3)
- [Linux 内核 5.10.47](https://hub.docker.com/layers/docker/for-desktop-kernel/5.10.47-0b705d955f5e283f62583c4e227d64a7924c138f/images/sha256-a4c79bc185ec9eba48dcc802a8881b9d97e532b3f803d23e5b8d4951588f4d51?context=repo)

### Bug 修复与小幅改进

- 更新内核配置以修复 [Docker Desktop 3.0.0](#docker-desktop-300) 中的性能回归问题，该问题导致容器端口发布耗时比旧版本长 10 倍。更多信息请参阅 [linuxkit/linuxkit#3701](https://github.com/linuxkit/linuxkit/pull/3701) 和 [docker/for-mac#5668](https://github.com/docker/for-mac/issues/5668)。
- 修复 DNS 服务器在接收到异常大数据报时失败的问题。
- 修复 iptables 更新时产生虚假跟踪记录的问题。
- 修复添加多个端口转发选项时速度缓慢的问题。
- 修复 WSL 2 同步代码在 WSL 2 主目录与 Windows 主目录相同时创建悬空符号链接的问题。修复了 [docker/for-win#11668](https://github.com/docker/for-win/issues/11668)。
- 修复从 3.5.x 升级后，当 Linux WSL 2 主目录与 Windows 主目录相同时 `docker context ls` 命令异常的问题。
- 修复 `%PROGRAMDATA%\Docker` 目录权限问题，避免潜在的 Windows 容器安全风险。参见 [CVE-2021-37841](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-37841)。感谢 [Alessio Dalla Piazza](http://it.linkedin.com/in/alessiodallapiazza) 发现此问题，以及 @kevpar 的有益讨论。
- 修复 WSL 2 下 Linux 主目录被错误设置为 Windows 主目录（如 `/mnt/c/Users/...`）的问题。
- 修复桌面应用因无法解析 CLI 上下文而无法启动的问题。修复了 [docker/for-win#11601](https://github.com/docker/for-win/issues/11601)。
- 修复容器内日志显示相关问题 [docker/for-win#11251](https://github.com/docker/for-win/issues/11251)。
- 修复 Windows 后台智能传输服务失败导致 Docker Desktop 无法启动的问题。[docker/for-win#11273](https://github.com/docker/for-win/issues/11273)

## Docker Desktop 3.5.2
2021-07-08

### 新增功能

**开发环境预览版**：开发环境功能让您无需在不同 Git 分支间切换，即可与团队成员无缝协作并将代码部署到成员机器上。使用开发环境时，只需一键即可与团队成员共享未完成的工作，且无需处理任何合并冲突。

### 升级组件

- [Compose V2 beta 6](https://github.com/docker/compose-cli/releases/tag/v2.0.0-beta.6)
  - `compose run` 和 `compose exec` 命令现在为标准输出和标准错误使用独立流。参见 [docker/compose-cli#1873](https://github.com/docker/compose-cli/issues/1873)。
  - `compose run` 和 `compose exec` 命令支持分离键。修复了 [docker/compose-cli#1709](https://github.com/docker/compose-cli/issues/1709)。
  - 修复 `compose rm` 命令的 `--force` 和 `--volumes` 标志。参见 [docker/compose-cli#1844](https://github.com/docker/compose-cli/issues/1844)。
  - 修复网络的 IPAM 配置，服务现在可定义固定 IP。修复了 [docker/compose-cli#1678](https://github.com/docker/compose-cli/issues/1678) 和 [docker/compose-cli#1816](https://github.com/docker/compose-cli/issues/1816)。

- 开发环境
  - 支持 VS Code Insiders 版本。
  - 允许用户在克隆项目时指定分支。参见 [dev-environments#11](https://github.com/docker/dev-environments/issues/11)

### Bug 修复与小幅改进

- 开发环境：修复某些创建和删除场景下的空白屏幕问题。修复了 [dev-environments#4](https://github.com/docker/dev-environments/issues/4)
- 开发环境：修复删除环境时的错误处理。修复了 [dev-environments#8](https://github.com/docker/dev-environments/issues/8)
- 开发环境：在环境创建或删除过程中，**启动**、**停止**和**共享**按钮将被禁用。
- 应用启动或 Windows/Linux 容器切换时不再自动切换 CLI 上下文。修复了 [docker/for-mac#5787](https://github.com/docker/for-mac/issues/5787) 和 [docker/for-win#11530](https://github.com/docker/for-win/issues/11530)。
- 修复 iptables 更新时产生虚假跟踪记录的问题。
- 修复添加多个端口转发选项时的延迟问题。

## Docker Desktop 3.5.1
2021-06-25

### 新增功能

**开发环境预览版**：开发环境功能让您无需在不同 Git 分支间切换，即可与团队成员无缝协作并将代码部署到成员机器上。使用开发环境时，只需一键即可与团队成员共享未完成的工作，且无需处理任何合并冲突。

**Compose V2 beta 版**：Docker Desktop 现在包含 Compose V2 的 beta 版本，支持作为 Docker CLI 一部分的 `docker compose` 命令。虽然 `docker-compose` 仍受支持并维护，但 Compose V2 实现直接依赖于 compose-go 绑定（作为规范的一部分进行维护）。Docker CLI 中的 compose 命令支持大多数 `docker-compose` 命令和标志，预期可完全替代 `docker-compose`。仍有少数标志尚未实现，有关新 compose 命令支持的标志信息，请参阅 docker-compose 兼容性列表。如果在使用 Compose V2 时遇到问题，您可以通过修改 Docker Desktop **实验性**设置，或运行命令 `docker-compose disable-v2` 轻松切换回 Compose v1。请在 [Compose-CLI](https://github.com/docker/compose-cli/issues) GitHub 仓库中创建 issue 来反馈您对新的 'compose' 命令的使用体验。

### Bug 修复与小幅改进

- 修复当临时文件夹路径包含点号时用户无法安装 Docker Desktop 的问题。修复了 [docker/for-win#11514](https://github.com/docker/for-win/issues/11514)
- 修复指向 Docker 处理上传诊断数据策略详情链接的问题。修复了 [docker/for-mac#5741](https://github.com/docker/for-mac/issues/5741)

- 修复运行 `docker login` 命令时出现 credStore 超时错误的问题。修复了 [docker/for-win#11472](https://github.com/docker/for-win/issues/11472)
- Docker Desktop 现在允许 WSL 2 集成代理在 `/etc/wsl.conf` 格式错误时仍能启动。

- 修复因 Hyper-V 虚拟机过早重启导致断电后 Docker Desktop 无法重启的问题。
- 默认 `docker` CLI `context` 现在在 Linux 容器模式下为 `desktop-linux`，在 Windows 容器模式下为 `desktop-windows`。
- Docker Desktop 反馈弹窗现在仅在点击 Docker 菜单时显示。

## Docker Desktop 3.4.0
2021-06-09

### 新增功能

**卷管理**：Docker Desktop 用户现在可以通过 Docker Dashboard 创建和删除卷，并查看哪些卷正在被使用。更多信息请参阅[浏览卷](../use-desktop/volumes.md)。

**Compose V2 beta 版**：Docker Desktop 现在包含 Compose V2 的 beta 版本，支持作为 Docker CLI 一部分的 `docker compose` 命令。虽然 `docker-compose` 仍受支持并维护，但 Compose V2 实现直接依赖于 compose-go 绑定（作为规范的一部分进行维护）。Docker CLI 中的 compose 命令支持大多数 `docker-compose` 命令和标志，预期可完全替代 `docker-compose`。仍有少数标志尚未实现，有关新 compose 命令支持的标志信息，请参阅 docker-compose 兼容性列表。如果在使用 Compose V2 时遇到问题，您可以通过修改 Docker Desktop **实验性**设置，或运行命令 `docker-compose disable-v2` 轻松切换回 Compose v1。请在 [Compose-CLI](https://github.com/docker/compose-cli/issues) GitHub 仓库中创建 issue 来反馈您对新的 'compose' 命令的使用体验。

## Docker Desktop 3.3.0
2021-04-08

### 新增功能

您现在可以指定何时下载和安装 Docker Desktop 更新。当有更新可用时，Docker Desktop 会显示一个图标指示新版本的可用性。您可以随时在后台下载更新。下载完成后，只需点击"更新并重启"即可安装最新更新。

专业开发用途的 Docker Desktop 用户有时可能需要跳过特定更新。因此，付费订阅用户可以在提醒出现时跳过特定更新的通知。

对于 IT 管理环境中没有管理员权限安装 Docker Desktop 更新的开发人员，如果您的 Docker ID 属于团队订阅，现在设置菜单中有一个选项可以完全退出 Docker Desktop 更新的通知。

## Docker Desktop 3.0.0
2020-12-10

## 新增功能

- Docker Desktop 版本现在使用三位数字编号。
- Docker Desktop 更新现在更小，因为将使用增量补丁应用。更多信息请参阅。
- `docker compose` 的首个版本（作为现有 `docker-compose` 的替代）。支持一些基本命令，但尚不具备 `docker-compose` 的完整功能。

  - 支持以下子命令：`up`、`down`、`logs`、`build`、`pull`、`push`、`ls`、`ps`
  - 支持基本卷、绑定挂载、网络和环境变量

    请在 [compose-cli](https://github.com/docker/compose-cli/issues) GitHub 仓库中创建 issue 来反馈您的使用体验。
