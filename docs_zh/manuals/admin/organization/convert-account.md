{{< summary-bar feature_name="Admin orgs" >}}

了解如何将现有的用户账户转换为组织。如果您需要多个用户访问您的账户及其关联的仓库，此功能非常有用。将其转换为组织后，您可以通过[团队](/manuals/admin/organization/manage-a-team.md)和[角色](/manuals/enterprise/security/roles-and-permissions.md)更好地控制这些用户的权限。

将用户账户转换为组织时，该账户默认会迁移到 Docker Team 订阅。

## 先决条件

在将用户账户转换为组织之前，请确保满足以下要求：

- 您要转换的用户账户不得是公司或任何团队或组织的成员。您必须将该账户从所有团队、组织或公司中移除。

    操作步骤如下：
    1. 导航到 **My Hub**，然后选择您需要离开的组织。
    2. 在 **Members** 选项卡中找到您的用户名。
    3. 选择 **More options** 菜单，然后选择 **Leave organization**。

    如果该用户账户是任何组织或公司的唯一所有者，请先将另一个用户指定为所有者角色，然后再将自己从组织或公司中移除。

- 您必须准备一个单独的 Docker ID，以便在转换过程中将其指定为组织的所有者。

    如果您想将用户账户转换为组织账户，但没有其他用户账户，则需要创建一个新的用户账户，并将其指定为新组织的所有者。拥有所有者角色后，该用户账户将拥有完全的管理权限来配置和管理组织。您可以在转换后为更多用户分配所有者角色。

## 转换账户时会发生什么

将账户转换为组织时，会发生以下情况：

- 此过程会删除账户的电子邮件地址。通知将改为发送给组织所有者。转换后，您可以将删除的电子邮件地址用于另一个账户。
- 当前订阅将自动取消，您的新订阅将开始。
- 仓库命名空间和名称不会更改，但转换账户会移除所有仓库协作者。转换账户后，您需要将仓库协作者添加为团队成员。
- 现有的自动构建将显示为由添加到组织的第一个所有者设置。
- 您添加为第一个所有者的用户账户将拥有完全的管理权限来配置和管理组织。
- 要将用户的个人访问令牌 (PAT) 转移到您转换后的组织，您必须将该用户指定为组织所有者。这将确保与该用户账户关联的任何 PAT 都转移到组织所有者。

## 将账户转换为组织

> [!IMPORTANT]
>
> 将账户转换为组织是永久性的。请备份您想要保留的任何数据或设置。

1. 登录 [Docker Home](https://app.docker.com/)。
2. 选择右上角的头像以打开下拉菜单。
3. 在 **Account settings** 中，选择 **Convert**。
4. 查看有关转换用户账户的警告。此操作无法撤销，并且对您的资产和账户有重大影响。
5. 输入 **Username of new owner** 以设置组织所有者。您指定的新 Docker ID 将成为组织的所有者。您不能使用与您尝试转换的账户相同的 Docker ID。
6. 选择 **Confirm**。新所有者将收到一封通知电子邮件。使用该所有者账户登录并管理新组织。
