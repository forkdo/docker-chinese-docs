了解如何在 Docker Hub 和 Docker Admin Console 中管理组织成员。

## 邀请成员

所有者可以通过 Docker ID、电子邮件地址或包含电子邮件地址的 CSV 文件邀请新成员加入组织。如果被邀请者没有 Docker 帐户，他们必须先创建帐户并验证电子邮件地址，然后才能接受加入组织的邀请。邀请成员时，其待处理的邀请会占用一个席位。

### 通过 Docker ID 或电子邮件地址邀请成员

请按照以下步骤通过 Docker ID 或电子邮件地址邀请成员加入组织：

1. 登录到 [Docker Home](https://app.docker.com) 并从左上角的帐户下拉菜单中选择您的组织。
1. 选择 **成员**，然后选择 **邀请**。
1. 选择 **电子邮件或用户名**。
1. 按照屏幕上的说明邀请成员。最多可邀请 1000 名成员，多个条目之间用逗号、分号或空格分隔。

> [!NOTE]
>
> 邀请成员时，您需要为其分配角色。有关每个角色的访问权限详细信息，请参阅 [角色和权限](/manuals/enterprise/security/roles-and-permissions.md)。

待处理的邀请将显示在表格中。被邀请者会收到一封电子邮件，其中包含指向 Docker Hub 的链接，他们可以在该链接处接受或拒绝邀请。

### 通过 CSV 文件邀请成员

要通过包含电子邮件地址的 CSV 文件邀请多名成员加入组织：

1. 登录到 [Docker Home](https://app.docker.com) 并从左上角的帐户下拉菜单中选择您的组织。
1. 选择 **成员**，然后选择 **邀请**。
1. 选择 **CSV 上传**。
1. 可选：选择 **下载模板 CSV 文件** 以下载示例 CSV 文件。以下是有效 CSV 文件内容的示例：

```text
email
docker.user-0@example.com
docker.user-1@example.com
```

CSV 文件要求：

- 文件必须包含标题行，至少包含一个名为 email 的标题。允许额外的列，但在导入时会被忽略。
- 文件最多可包含 1000 个电子邮件地址（行）。要邀请超过 1000 名用户，请创建多个 CSV 文件，并对每个文件执行本任务中的所有步骤。

1. 创建新的 CSV 文件或从其他应用程序导出 CSV 文件。

- 要从其他应用程序导出 CSV 文件，请参阅该应用程序的文档。
- 要创建新的 CSV 文件，请在文本编辑器中打开新文件，在第一行键入 email，在后续行中每行键入一个用户电子邮件地址，然后将文件保存为 .csv 扩展名。

1. 选择 **浏览文件**，然后选择您的 CSV 文件，或者将 CSV 文件拖放到 **选择要上传的 CSV 文件** 框中。您一次只能选择一个 CSV 文件。

> [!NOTE]
>
> 如果您的 CSV 文件中的电子邮件地址数量超过组织中可用席位的数量，您将无法继续邀请成员。要邀请成员，您可以购买更多席位，或者从 CSV 文件中删除一些电子邮件地址并重新选择新文件。要购买更多席位，请参阅 [为订阅添加席位](/manuals/subscription/manage-seats.md) 或 [联系销售](https://www.docker.com/pricing/contact-sales/)。

1. CSV 文件上传后，选择 **审核**。

有效的电子邮件地址以及存在任何问题的电子邮件地址将显示出来。电子邮件地址可能有以下问题：

- 无效的电子邮件：电子邮件地址不是有效地址。如果您发送邀请，该电子邮件地址将被忽略。您可以在 CSV 文件中更正电子邮件地址并重新导入文件。
- 已邀请：用户已收到邀请邮件，不会再次发送邀请邮件。
- 成员：用户已经是您组织的成员，不会发送邀请邮件。
- 重复：CSV 文件中多次出现同一电子邮件地址。只会向该用户发送一封邀请邮件。

1. 按照屏幕上的说明邀请成员。

> [!NOTE]
>
> 邀请成员时，您需要为其分配角色。有关每个角色的访问权限详细信息，请参阅 [角色和权限](/manuals/enterprise/security/roles-and-permissions.md)。

待处理的邀请将显示在表格中。被邀请者会收到一封电子邮件，其中包含指向 Docker Hub 的链接，他们可以在该链接处接受或拒绝邀请。

### 通过 API 邀请成员

您可以使用 Docker Hub API 批量邀请成员。有关详细信息，请参阅 [批量创建邀请](https://docs.docker.com/reference/api/hub/latest/#tag/invites/paths/~1v2~1invites~1bulk/post) API 端点。

## 接受邀请

当邀请发送到用户的电子邮件地址时，他们会收到一封包含指向 Docker Hub 链接的电子邮件，他们可以在该链接处接受或拒绝邀请。要接受邀请：

1. 检查您的电子邮件收件箱并打开 Docker 发来的邀请加入 Docker 组织的邮件。
1. 要打开指向 Docker Hub 的链接，请选择 **点击此处** 链接。

   > [!WARNING]
   >
   > 邀请电子邮件链接在 14 天后过期。如果您的邮件链接已过期，您可以使用发送链接的电子邮件地址登录到 [Docker Hub](https://hub.docker.com/)，然后从 **通知** 面板接受邀请。

1. 将打开 Docker 创建帐户页面。如果您已有帐户，请选择 **已有帐户？登录**。
如果您还没有帐户，请使用您收到邀请的电子邮件地址创建一个帐户。
1. 可选：如果您没有帐户并且刚刚创建了一个帐户，您必须导航回电子邮件收件箱，并使用 Docker 验证邮件验证您的电子邮件地址。
1. 登录到 Docker Hub 后，从顶级导航菜单中选择 **我的 Hub**。
1. 在您的邀请上选择 **接受**。

接受邀请后，您现在是该组织的成员。

## 管理邀请

邀请成员后，您可以根据需要重新发送或删除邀请。

### 重新发送邀请

您可以在 Admin Console 中单独发送邀请，或批量重新发送邀请。

要重新发送单个邀请：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **成员**。
1. 选择被邀请者旁边的 **操作菜单**，然后选择 **重新发送**。
1. 选择 **邀请** 以确认。

要批量重新发送邀请：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **成员**。
1. 使用 **用户名** 旁边的 **复选框** 批量选择用户。
1. 选择 **重新发送邀请**。
1. 选择 **重新发送** 以确认。

### 删除邀请

要从 Admin Console 中删除邀请：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **成员**。
1. 选择被邀请者旁边的 **操作菜单**，然后选择 **删除被邀请者**。
1. 选择 **删除** 以确认。

## 管理团队成员

使用 Docker Hub 或 Admin Console 添加或删除团队成员。组织所有者可以将成员添加到组织内的一个或多个团队。

### 将成员添加到团队

要使用 Admin Console 将成员添加到团队：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **团队**。
1. 选择团队名称。
1. 选择 **添加成员**。您可以通过搜索其电子邮件地址或用户名来添加成员。

   > [!NOTE]
   >
   > 被邀请者必须先接受邀请加入组织，然后才能被添加到团队。

### 从团队中删除成员

> [!NOTE]
>
> 如果您的组织使用启用了 [SCIM](/manuals/enterprise/security/provisioning/scim.md) 的单点登录 (SSO)，您应该从身份提供商 (IdP) 中删除成员。这将自动从 Docker 中删除成员。如果 SCIM 已禁用，您必须在 Docker 中手动管理成员。

组织所有者可以在 Docker Hub 或 Admin Console 中从团队中删除成员。从团队中删除成员将撤销他们对允许资源的访问权限。

要使用 Admin Console 从特定团队中删除成员：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **团队**。
1. 选择团队名称。
1. 选择用户姓名旁边的 **X** 将其从团队中删除。
1. 出现提示时，选择 **删除** 以确认。

### 更新成员角色

组织所有者可以管理组织内的 [角色](/security/for-admins/roles-and-permissions/)。如果组织是公司的一部分，公司所有者也可以管理该组织的角色。如果您启用了 SSO，您可以使用 [SCIM 进行角色映射](/security/for-admins/provisioning/scim/)。

> [!NOTE]
>
> 如果您是组织的唯一所有者，您需要先分配新所有者，然后才能编辑您的角色。

要在 Admin Console 中更新成员角色：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **成员**。
1. 找到您要编辑角色的成员的用户名。选择 **操作** 菜单，然后选择 **编辑角色**。

## 导出成员 CSV 文件

{{< summary-bar feature_name="Admin 组织" >}}

所有者可以导出包含所有成员的 CSV 文件。公司的 CSV 文件包含以下字段：

- 姓名：用户的姓名
- 用户名：用户的 Docker ID
- 电子邮件：用户的电子邮件地址
- 所属组织：用户在公司内所属的所有组织
- 受邀组织：用户在公司内受邀加入的所有组织
- 帐户创建时间：用户帐户创建的时间和日期

要导出成员的 CSV 文件：

1. 登录到 [Docker Home](https://app.docker.com/) 并选择您的组织。
1. 选择 **成员**。
1. 选择 **下载** 图标以导出所有成员的 CSV 文件。
