本文将指导您完成所有者和成员的入职流程，并介绍如何使用 SSO 和 SCIM 等工具为未来的入职流程提供可扩展的解决方案。

## 邀请所有者

创建 Docker 组织时，您会自动成为该组织的唯一所有者。虽然这一步是可选的，但添加额外的所有者可以显著简化组织的入职和管理流程，通过分散管理职责来提高效率。同时，这也确保了组织的连续性，避免因主要所有者不可用而导致的工作阻塞。

有关所有者的详细信息，请参阅 [角色和权限](/manuals/enterprise/security/roles-and-permissions.md)。

## 邀请成员并分配角色

成员被授予对资源的受控访问权限，并享受增强的组织级权益。当您邀请成员加入 Docker 组织时，需要立即为其分配一个角色。

### 邀请成员的优势

- 增强的可见性：深入了解用户活动，便于监控访问并执行安全策略。
- 简化协作：通过授予成员对共享资源和仓库的访问权限，帮助他们高效协作。
- 改进的资源管理：在组织内组织和跟踪用户，确保资源的最优分配。
- 访问增强功能：成员可享受组织级权益，例如更高的拉取限制和访问 Docker 高级功能。
- 安全控制：在组织级别应用和执行安全设置，降低未受管账户相关的安全风险。

详细信息，请参阅 [管理组织成员](/manuals/admin/organization/members.md)。

## 构建面向未来的用户管理

构建强大且面向未来的用户管理方案需要结合自动化配置、集中身份认证和动态访问控制。实施这些实践可确保环境具备可扩展性、安全性和高效性。

### 使用单点登录 (SSO) 实现安全身份认证

将 Docker 与您的身份提供商集成，可以简化用户访问并增强安全性。

SSO 具有以下优势：

- 简化登录流程：用户使用组织凭据登录。
- 减少与密码相关的安全漏洞。
- 简化入职流程：与 SCIM 和组映射无缝配合，实现自动化配置。

更多信息，请参阅 [SSO 文档](/manuals/enterprise/security/single-sign-on/_index.md)。

### 使用 SCIM 和 JIT 配置自动化入职流程

通过 [SCIM](/manuals/enterprise/security/provisioning/scim.md) 和 [即时 (JIT) 配置](/manuals/enterprise/security/provisioning/just-in-time.md) 简化用户配置和角色管理。

使用 SCIM，您可以：

- 与身份提供商自动同步用户和角色。
- 根据目录变更自动添加、更新或删除用户。

使用 JIT 配置，您可以：

- 基于 [组映射](#simplify-access-with-group-mapping) 在用户首次登录时自动添加用户。
- 通过消除预邀请步骤来减少管理开销。

### 使用组映射简化访问

组映射通过将身份提供商的组与 Docker 角色和团队关联，实现权限管理的自动化。

它还能够：

- 减少角色分配中的人工错误。
- 确保一致的访问控制策略。
- 帮助您在团队增长或变化时扩展权限管理。

有关工作原理的更多信息，请参阅 [组映射](/manuals/enterprise/security/provisioning/group-mapping.md)。
