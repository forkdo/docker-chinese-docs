容器提供了一种一致的方式来构建、共享和在不同环境中运行应用程序。虽然容器通常用于将应用程序容器化，但它们也使得运行开发所需的关键服务变得异常简单。你不再需要安装或连接到远程数据库，而是可以轻松启动自己的数据库。但可能性远不止于此。

通过容器化支持开发，你可以使用容器来增强开发环境，通过模拟或运行应用程序所需服务的本地实例。这提供了更快的反馈循环，减少与远程服务的耦合，并增强了测试错误状态的能力。

最重要的是，无论你正在开发的主要应用程序是否运行在容器中，你都可以获得这些好处。

## 你将学到的内容

- 容器化支持开发的含义
- 如何将非容器化应用程序连接到容器化服务
- 使用容器模拟或运行服务本地实例的几个示例
- 如何使用容器为开发环境添加额外的故障排除和调试工具

## 适用人群

- 希望减少与共享或已部署基础设施或远程 API 端点耦合的团队
- 希望减少在开发过程中直接使用云服务的复杂性和成本的团队
- 希望更轻松地可视化数据库、队列等内容的开发者
- 希望减少设置开发环境复杂性，同时不影响应用程序开发本身的团队

## 工具集成

与 Docker Compose 和 Testcontainers 配合良好。

## 模块

### 什么是容器化支持开发？

容器化支持开发是指使用容器来增强你的开发环境，通过运行应用程序所依赖服务的本地实例或模拟器。一旦你开始使用容器，就很容易添加额外的服务来可视化或排查服务中的问题。

### 演示：在本地开发云端

在开发应用程序时，通常更容易将应用程序的某些方面外包给云服务，例如 Amazon S3。然而，在本地开发中连接到这些服务会引入 IAM 策略、网络约束和配置复杂性。虽然这些要求在生产环境中很重要，但它们显著增加了开发环境的复杂性。

通过容器化支持开发，你可以在开发和测试期间运行这些服务的本地实例，从而无需复杂的设置。在此演示中，你将看到 LocalStack 如何让你能够完全从开发者工作站轻松开发和测试应用程序。
