"软件供应链"一词指的是软件开发和交付的端到端流程，从开发到部署和维护。软件供应链安全，简称"S3C"，是保护供应链中组件和流程的实践。

S3C是组织处理软件安全的根本性变革。传统上，在软件行业中，安全和合规大多被当作事后考虑，留到软件交付或发布阶段。而S3C将安全集成到整个软件开发生命周期中，从开发和测试的内循环，到发布和监控的外循环。

遵循软件供应链安全的最佳实践非常重要，因为它帮助组织保护软件免受安全威胁、合规风险和其他漏洞的影响。实施软件供应链安全框架可以提高项目在各利益相关者之间的可见性、协作性和可追溯性。这有助于组织更有效地检测、响应和修复威胁。

## 保护软件供应链

构建安全的软件供应链涉及几个关键步骤，例如：

- 识别用于构建和运行应用程序的软件组件和依赖项。
- 在整个软件开发生命周期中自动化安全测试。
- 监控软件供应链中的安全威胁。
- 实施安全管理策略，规范软件的构建方式及其包含的组件。

管理软件供应链是一项复杂的任务，尤其是在当今软件使用来自不同来源的多个组件构建的时代。组织需要清楚了解他们使用的软件组件以及相关的安全风险。

## Docker Scout 的独特之处

Docker Scout 是一个旨在帮助组织保护软件供应链的平台。它提供工具和服务，用于识别和管理软件资产和策略，以及自动修复安全威胁。

与传统安全工具专注于在软件开发生命周期的特定阶段进行定时扫描不同，Docker Scout 使用覆盖整个软件供应链的现代事件驱动模型。这意味着当披露影响您镜像的新漏洞时，您的更新风险评估在几秒内即可获得，并且在开发过程的早期阶段就能发现。

Docker Scout 通过分析镜像的组成来创建软件物料清单（SBOM）。SBOM 与安全公告进行交叉引用，以识别影响您镜像的 CVE。Docker Scout 集成了[20 多个不同的安全公告源](/manuals/scout/deep-dive/advisory-db-sources.md)，并实时更新其漏洞数据库。这确保您的安全状况始终使用最新可用信息表示。
