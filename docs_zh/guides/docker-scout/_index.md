当容器镜像不安全时，可能会引发重大风险。约有 60% 的组织报告称，一年内至少经历过一次安全漏洞或事件，[导致业务运营中断][CSA]。这些事件通常会造成显著的停机时间，44% 的受影响公司每次事件停机超过一小时。财务影响巨大，[平均数据泄露成本达到 445 万美元][IBM]。这凸显了维护强大容器安全措施的关键重要性。

Docker Scout 通过提供自动化的漏洞检测和修复、解决不安全的容器镜像问题以及确保符合安全标准，来增强容器安全性。

[CSA]: https://cloudsecurityalliance.org/blog/2023/09/21/2023-global-cloud-threat-report-cloud-attacks-are-lightning-fast
[IBM]: https://www.ibm.com/reports/data-breach

## 您将学到的内容

- 定义安全软件供应链 (SSSC)
- 查看 SBOM 及其使用方法
- 检测和监控漏洞

## 工具集成

与 Docker Desktop、GitHub Actions、Jenkins、Kubernetes 和其他 CI 解决方案完美集成。

## 适用人群

- 需要在 CI/CD 管道中集成自动化安全检查的 DevOps 工程师，以增强其工作流的安全性和效率。
- 希望使用 Docker Scout 在开发早期识别和修复漏洞的开发人员，确保生成安全的容器镜像。
- 必须执行安全合规、进行漏洞评估并确保容器化应用程序整体安全的安全专业人员。
