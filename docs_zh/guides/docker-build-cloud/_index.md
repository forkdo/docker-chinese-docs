98% 的开发者每天花费多达一小时等待构建完成（[Incredibuild: 2022 Big Dev Build Times](https://www.incredibuild.com/survey-report-2022)）。
繁重、复杂的构建可能成为开发团队的主要障碍，拖慢本地开发和 CI/CD 流水线的速度。

<!-- vale Vale.Spelling = YES -->

Docker Build Cloud 通过加速镜像构建时间来提高开发人员效率，减少挫败感，并帮助您缩短发布周期。

## 适用人群

- 希望解决镜像构建缓慢常见原因的任何人：本地资源有限、模拟速度慢以及团队间缺乏构建协作。
- 在较旧机器上工作的开发者，希望加快构建速度。
- 在同一仓库上协作的开发团队，希望通过共享缓存减少等待时间。
- 执行多架构构建的开发者，不希望花费数小时配置和为模拟器重建。

## 您将学到

- 在本地和 CI 中更快地构建容器镜像
- 加速多平台镜像的构建
- 复用预构建镜像以加速工作流

## 工具集成

与 Docker Compose、GitHub Actions 和其他 CI 解决方案完美配合
