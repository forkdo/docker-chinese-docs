98% 的开发者每天花费长达一个小时等待构建完成
([Incredibuild: 2022 Big Dev Build Times](https://www.incredibuild.com/survey-report-2022))。
繁重、复杂的构建会成为开发团队的主要障碍，减慢本地开发和 CI/CD 流程的速度。

<!-- vale Vale.Spelling = YES -->

Docker Build Cloud 加快了镜像构建速度，从而提高了开发人员的工作效率，减少了挫败感，并帮助您缩短发布周期。

## 适合人群

- 任何想要解决导致镜像构建缓慢的常见原因的人：本地资源有限、仿真速度慢以及缺乏团队间的构建协作。
- 使用旧机器但希望更快构建的开发人员。
- 在同一仓库上工作的开发团队，希望通过共享缓存来减少等待时间。
- 执行多架构构建但不想花费数小时配置仿真器和重新构建的开发人员。

## 您将学到什么

- 在本地和 CI 中更快地构建容器镜像
- 加速多平台镜像的构建
- 重用预构建镜像以加快工作流程

## 工具集成

与 Docker Compose、GitHub Actions 和其他 CI 解决方案配合良好
